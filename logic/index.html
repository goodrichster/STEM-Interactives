Quantifiers & Domains

Learn about universal (∀) and existential (∃) quantifiers, domains (ℕ, ℤ, ℚ, ℝ), and how to translate between English and symbolic logic.

Key Concepts:

Universal Quantifier (∀): "For all" - asserts that a property holds for every element in a domain.

Existential Quantifier (∃): "There exists" - asserts that a property holds for at least one element in a domain.

"No" Statements: "No x..." translates to ∀x, ¬(...)

Domains: The set over which quantifiers range (ℕ = Natural Numbers, ℤ = Integers, ℚ = Rationals, ℝ = Reals).

Symbolic Translation Examples:
"All integers are even" → ∀x ∈ ℤ, x is even
"Some real numbers are negative" → ∃x ∈ ℝ, x < 0
"No natural numbers are negative" → ∀x ∈ ℕ, x ≥ 0

Negation Wizard

Learn how to correctly negate quantified statements by switching the quantifier AND negating the predicate. Avoid common mistakes like swapping quantifiers without negating predicates.

De Morgan's Laws for Quantifiers:

¬∀x ∈ D, P(x) ≡ ∃x ∈ D, ¬P(x)

¬∃x ∈ D, P(x) ≡ ∀x ∈ D, ¬P(x)

Common Mistakes to Avoid:

❌ ¬∀x ∈ D, P(x) ≡ ∃x ∈ D, P(x) (Wrong: swapped quantifier but didn't negate predicate)

✅ ¬∀x ∈ D, P(x) ≡ ∃x ∈ D, ¬P(x) (Correct: swapped quantifier AND negated predicate)

Negation Process:
Step 1: Switch the quantifier (∀ ↔ ∃)
Step 2: Negate the predicate (P(x) → ¬P(x))
Step 3: Verify the result makes logical sense

Counterexample Builder

Learn how to prove universal statements false with a single counterexample, and how to validate existential statements with examples.

Universal Statements (∀): Require one counterexample to be proven false. All cases must be true to be proven true.

Existential Statements (∃): Require one example to be proven true. All cases must be false to be proven false.

"No" Statements: Equivalent to universal statements (¬∃x ≡ ∀x,¬). One counterexample proves them false.

Counterexample Logic:
To disprove "∀x ∈ D, P(x)": Find one element a ∈ D such that ¬P(a)
To prove "∃x ∈ D, P(x)": Find one element a ∈ D such that P(a)
To disprove "No x ∈ D, P(x)": Find one element a ∈ D such that P(a)

Conditionals & Equivalence

Master conditional statements (p → q), biconditionals (p ↔ q), and logical equivalence through truth tables and transformations.

Conditional (p → q): False only when p is true and q is false. Equivalent to ¬p ∨ q.

Biconditional (p ↔ q): True when both p and q have the same truth value.

Logical Equivalence: Two statements are equivalent if they have identical truth values in all cases.

Key Equivalences:
p → q ≡ ¬p ∨ q (Conditional as disjunction)
p → q ≡ ¬q → ¬p (Contrapositive)
¬(p → q) ≡ p ∧ ¬q (Negation of conditional)
p ↔ q ≡ (p → q) ∧ (q → p) (Biconditional as conjunction)

Laws of Reasoning

Apply fundamental laws of logic including Law of Detachment, Simplification, Addition, Disjunctive Syllogism, and Law of Syllogism.

Law of Detachment: From p → q and p, conclude q. Requires both the conditional and its antecedent.

Simplification: From p ∧ q, conclude p (or q).

Addition: From p, conclude p ∨ q.

Disjunctive Syllogism: From p ∨ q and ¬p, conclude q.

Law of Syllogism: From p → q and q → r, conclude p → r.

Law Preconditions & Conclusions:
Detachment: p → q, p ⊢ q
Simplification: p ∧ q ⊢ p
Addition: p ⊢ p ∨ q
Disjunctive Syllogism: p ∨ q, ¬p ⊢ q
Syllogism: p → q, q → r ⊢ p → r

Statement–Reason Proofs

Build formal proofs line-by-line with justified reasoning. Each statement must follow logically from previous statements using valid laws.

Proof Structure: Two-column format with Statements (left) and Reasons (right).

Reason Requirements: Each reason must cite prior lines and a valid logical law.

Dependency Validation: Each statement can only reference previous lines in the proof.

Proof Validation Rules:
1. Each statement must follow logically from prior statements
2. Each reason must cite valid law and correct line numbers
3. No forward referencing allowed
4. Conclusion must match goal statement
